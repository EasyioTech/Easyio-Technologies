# TypeScript Best Practices

TypeScript is a force multiplier. Used well, it catches bugs before production. Used poorly, it's verbose with little value.

## Core Principles

### 1. Strict Mode
Enable `strict: true` in tsconfig. Non-negotiable.

### 2. Avoid `any`
If you're writing `any`, pause and think. There's always a better type.

### 3. Use Generics
Reusable, type-safe abstractions. DRY principle applied to types.

### 4. Discriminated Unions
Better than optional fields. Exhaustive checking via discriminants.

## Common Patterns

```typescript
// Good: Discriminated union
type Result<T> = 
  | { status: 'success'; data: T }
  | { status: 'error'; error: Error };

// Bad: Optional fields everywhere
type Result<T> = { data?: T; error?: Error; status?: string };
```

## Tools

- `ts-unused-exports`: Find dead code
- `type-coverage`: Measure type safety coverage
- `tsc --noEmit`: Type checking without emit

## Conclusion

TypeScript is a long-term productivity investment. Strict types → fewer bugs → faster shipping.
